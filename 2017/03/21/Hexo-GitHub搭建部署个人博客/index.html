Hexo-GitHub搭建部署个人博客

发表于 2017-03-21 | 分类于 Hexo


　　在搭建Hexo博客并部署到GitHub的过程中，将基本流程和遇到的问题记录下来。

1、环境和准备工具

  • Windows 7/10
  • Git python
  • Node.js
  • GitHub

2、安装Git

  • https://git-scm.com/download/win
  • https://git-for-windows.github.io/

3、安装Node.js

　　选择安装地址PATH” E:\NodeJS “，接下来一直Next即可。

4、安装Hexo

　　利用npm命令安装Hexo，在Git python窗口中输入命令：

　　安装完成后，在Git python中，输入”$ npm - v”
　　此处的安装过程中，npm默认是国外的源，因为某些你懂的原因，速度极慢或超时报错。
　　解决方法是调整使用国内镜像源。推荐使用4.3方法编辑npm配置文件，永久使用国内镜像源。

5、创建Hexo文件夹

　　选择创建路径PATH “ E:\MyBlog “，执行（在文件夹内右键，选择Git python）”$ hexo init”，Hexo即会自动在目标文件夹内建立网站所需要的所有文件。

6、依赖包安装

　　在成功执行5&6两步后，在”E:\MyBlog”中初始化完成后的目录层次为：

  • _config.yml：网站的配置信息，可在此配置大部分参数；
  • package.json：应用程序的信息；
  • scaffolds：模板文件夹，当新建文章时Hexo会根据scaffold来建立文件；
  • source：资源文件夹存在用户次元。MarkDown和HTML文件会被解析并放到public文件夹内，而其他文件会被拷贝过去；
  • themes：网站主题文件夹，Hexo根据主题来生成静态页面。
